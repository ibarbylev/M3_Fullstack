# Методы изменения данных

### 🔧 Таблица: Методы изменения данных в QuerySet

| Метод                         | Что делает                                                | Пример                                                                                                               | Пояснение                                                                                |
| ----------------------------- | --------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------- |
| `update()`                    | Массовое обновление записей                               | `Book.objects.filter(author="Orwell").update(year_published=1950)`                                                   | Обновляет все записи, удовлетворяющие фильтру, **без вызова save()** и сигналов.         |
| `save()`                      | Сохраняет изменения одной записи                          | `book = Book.objects.get(id=1)  book.title = "New"  book.save()`                                                     | Используется для обновления **одной** записи, вызывает сигналы `pre_save`, `post_save`.  |
| `update_or_create()`          | Обновляет или создаёт запись                              | `Book.objects.update_or_create(  author="Huxley",  defaults={"title": "Brave New World", "year_published": 1932}  )` | Если запись с `author="Huxley"` есть, обновит. Если нет — создаст новую.                 |
| `bulk_update()` (Django 3.2+) | Массово обновляет сразу несколько объектов                | `Book.objects.bulk_update([b1, b2], ['title'])`                                                                      | Обновляет поля указанных объектов **одним SQL-запросом**. Эффективно при большом объёме. |
| `get_or_create()` + `save()`  | Создаёт или получает объект и вручную обновляет           | `book, created = Book.objects.get_or_create(author="Rowling")  book.title = "HP"  book.save()`                       | Если объект найден, можно его изменить и сохранить.                                      |
| `F()`                         | Обновление значений на основе текущих значений (SQL-side) | `Book.objects.update(year_published=F('year_published') + 1)`                                                        | Выполняет обновление **на уровне БД**, не загружая объекты в память.                     |

---

### 🔍 Пояснение по выбору метода

| Сценарий                                | Рекомендованный метод          |
| --------------------------------------- | ------------------------------ |
| Обновить **много записей** разом        | `update()` или `bulk_update()` |
| Обновить **одно поле** у одного объекта | `save()`                       |
| Увеличить число, не загружая объект     | `F()` + `update()`             |
| Обновить или создать объект             | `update_or_create()`           |
| Обновить список объектов в Python       | `bulk_update()`                |

---

### 🔁 Пример массового обновления `update()`

```python
Book.objects.filter(year_published__lt=2000).update(author="Unknown")
```

📌 Все книги, изданные до 2000 года, получат автора "Unknown".

---

### ⛏ Пример с `F()` — инкремент

```python
from django.db.models import F

Book.objects.all().update(year_published=F('year_published') + 1)
```

📌 Увеличит год публикации на 1 у всех книг.
