## 1️ Отображение списка объектов ([List Display](./theory_04__1_list_display.md))

Управляет тем, как объекты отображаются на странице списка:

* `list_display` – поля, отображаемые в таблице.
* `list_display_links` – поля, по которым можно перейти к редактированию.
* `list_editable` – поля, редактируемые прямо в списке.
* `list_filter` – фильтры по полям (например, даты, Boolean, ForeignKey).
* `search_fields` – поля, по которым работает поиск.
* `ordering` – сортировка объектов по умолчанию.
* `list_select_related` – оптимизация запросов через `select_related`.
* `list_prefetch_related` – оптимизация для `ManyToMany` и `reverse ForeignKey`.
* **Пагинация**:

  * `list_per_page` – количество объектов на странице (по умолчанию 100).
  * `list_max_show_all` – максимальное количество объектов при показе всех.

## 2️ Формы редактирования объектов ([Form Display](./theory_04__2_form_display.md))

Настройка формы создания и редактирования:

* `fields` – явный список полей формы.
* `exclude` – поля, которые не показывать.
* `fieldsets` – группировка полей с заголовками и описаниями.
* `readonly_fields` – поля только для чтения.
* `form` – использование кастомной формы `ModelForm`.
* `formfield_overrides` – замена виджетов для конкретных полей (например, TextField → Textarea).

---

## 3️ Инлайн-модели ([Inlines](./theory_04__3_inlines.md))

Редактирование связанных объектов прямо внутри формы родительской модели:

* `TabularInline` – компактная таблица.
* `StackedInline` – развернутый блок.
* `model` – связанная модель.
* `extra` – количество пустых форм для добавления новых объектов.
* `can_delete` – разрешение на удаление объектов.
* `min_num`, `max_num` – минимальное и максимальное количество инлайнов.
* `readonly_fields` – поля только для чтения в инлайне.

---

## 4️ Доступ и права ([Permissions](./theory_04__4_permissions.md))

Управление видимостью и возможностями пользователя:

* `has_add_permission` – можно ли добавлять объекты.
* `has_change_permission` – можно ли редактировать объекты.
* `has_delete_permission` – можно ли удалять объекты.
* `has_view_permission` – можно ли просматривать объекты.
* `get_queryset` – кастомизация набора объектов для конкретного пользователя.
* `get_readonly_fields` – динамическое управление доступом к полям.

---

## 5️ Поведение и кастомизация ([Behavior & Customization](./theory_04__5_behavior_and_customization.md))

Настройки интерфейса и удобства работы:

* **Автозаполнение и подготовка полей**:

  * `prepopulated_fields` – автозаполнение поля на основе другого.
  * `autocomplete_fields` – автозаполнение связей через поиск.

* **Действия и массовые операции**:

  * `actions` – список действий, доступных для выбранных объектов.
  * `actions_on_top`, `actions_on_bottom` – отображение кнопок действий сверху/снизу.

* **Сохранение и копирование объектов**:

  * `save_as` – кнопка «Сохранить как новый».
  * `save_on_top` – кнопки сохранения также вверху формы.

* **Оптимизация запросов**:

  * `list_select_related`, `list_prefetch_related` – уменьшение количества SQL-запросов при выводе связанных объектов.

* **Другие полезные настройки**:

  * `date_hierarchy` – навигация по датам.
  * `empty_value_display` – текст для пустых полей.
  * `radio_fields` – отображение полей как радио-кнопок вместо выпадающего списка.

---

## 6️ Кастомизация внешнего вида и поведения ([Templates, CSS, JS](./theory_04__6_customer_HTML_CSS_JS_1.md))

Позволяет изменять стандартный интерфейс админки и добавлять свой функционал:

* **Шаблоны**:

  * `change_form_template` – шаблон для формы редактирования объекта.
  * `change_list_template` – шаблон для страницы списка объектов.
  * `delete_confirmation_template` – шаблон для подтверждения удаления.
  * `object_history_template` – шаблон истории объекта.

* **Статические файлы (CSS и JS)**:

  * `Media` класс внутри `ModelAdmin` или формы:

    ```python
    class Media:
        css = {
            'all': ('my_styles.css',)
        }
        js = ('my_scripts.js',)
    ```
  * Можно использовать для валидации, динамических эффектов, кастомного поведения формы.

* **Дополнительная глобальная кастомизация**:

  * Переопределение встроенных шаблонов (`admin/change_form.html`, `admin/change_list.html`) для всего проекта.
  * Подключение CSS/JS через `admin.site.register()` или глобальные файлы.
