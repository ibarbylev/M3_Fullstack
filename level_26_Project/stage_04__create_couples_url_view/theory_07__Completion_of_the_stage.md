## Завершение этапа создания пар url-view

**Цель этапа** — сформировать и проверить базовую структуру маршрутизации всего проекта (без API).

### Задачи этапа:

1. **Создать все приложения**, необходимые для работы основного сайта (только каркас!).
2. **Настроить маршрутизацию** — обеспечить корректное прохождение запроса от `main/urls.py` до соответствующих `views` каждого приложения.
3. **Определить все представления (views)** — создать в каждом `views.py` классы на основе `Class-Based View` (только с заглушками, без логики).
4. **Проверить маршруты** — запустить базовый тест `test_urls_resolve`, который должен подтвердить, что:

   * каждый `url_name` успешно разрешается через `reverse()`;
   * результат `resolve()` указывает на правильный класс `view`.

После успешного прохождения теста `test_urls_resolve` можно считать, что:

* структура маршрутов сайта создана корректно;
* все `urls` связаны с соответствующими `views`;
* можно переходить к реализации логики и шаблонов во `views`.
